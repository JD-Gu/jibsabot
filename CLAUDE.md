# CLAUDE.md — 구대표집사봇 작업 규칙

## 1. 작업 기록
모든 작업 수행 후, 수행 내역과 결과를 `docs/tasks/YYYY-MM-DD.md` 파일에 시간순으로 기록한다.

- 파일이 없으면 새로 생성한다.
- 각 항목 형식: `HH:MM | 작업 내용 | 결과 (완료/실패/보류)`
- 코드 변경이 있으면 변경된 파일과 요약을 함께 기록한다.

## 2. 요구사항 확인
모든 작업 시작 전, `요구사항.md`를 먼저 읽고 현재 요구사항과 우선순위를 파악한다.

## 3. 기획서 참조
구현 방향, 채널 구조, 역할 정의, 로드맵 등 프로젝트 전반에 관한 판단이 필요할 때는 `기획서.md`를 참조한다.
