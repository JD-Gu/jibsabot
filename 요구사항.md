### S1. 지시 전달 및 결과 보고

**개요:** 대표님이 봇에게 지시하면, 해당 직원 DM 또는 채널에 내용을 전달하고, 회신이 오면 대표님께 결과 보고한다.

**시나리오 흐름:**
```
대표님 → 봇 DM/jj-프롬프트
  "김봉석 팀장한테 이번 주 금요일까지 HI-RTK 납품 일정 확인해달라고 해줘"

봇 → 김봉석 DM
  "안녕하세요 김봉석 팀장님, 구대표님 지시로 연락드립니다.
   이번 주 금요일까지 HI-RTK 납품 일정 확인 부탁드립니다. 회신 주시면 감사하겠습니다."

봇 → 대표님 DM (즉시)
  "✅ 김봉석 팀장님께 지시사항을 전달했습니다."

--- 이후 김봉석이 봇에게 회신 ---

김봉석 → 봇 DM
  "네, 확인했습니다. 금요일 오후 3시 출고 예정입니다."

봇 → 대표님 DM (회신 즉시)
  "📬 [지시 결과 보고]
   대상: 김봉석 팀장
   전달 내용: HI-RTK 납품 일정 확인 요청
   회신: '금요일 오후 3시 출고 예정입니다.'"
```

**구현 포인트:**
- 지시 컨텍스트(전달 대상 + 내용)를 메모리에 보관 → 회신 매칭
- 회신 감지: 봇 DM에 직원이 답장 시 대표님에게 포워딩
- 미회신 시 리마인더(예: 24시간 후) 여부 → 추후 설계

### S2. 직원 문의 처리 및 대표 컨펌

**개요:** 직원이 봇 DM에 문의하면, 간단한 건은 봇이 직접 대응하고, 중요한 건은 대표님 컨펌 후 회신한다. 처리 결과는 대표님께 보고.

**시나리오 흐름 (간단한 건):**
```
이지민 팀장 → 봇 DM
  "이번 주 월요일 회의 시간이 몇 시였나요?"

봇 (구글캘린더 조회 후 직접 답변)
  "안녕하세요 이지민 팀장님, 이번 주 월요일 회의는 오전 10시입니다."

봇 → 대표님 DM (처리 후 보고)
  "📋 [직원 문의 처리]
   문의자: 이지민 팀장
   내용: 이번 주 월요일 회의 시간 문의
   처리: 캘린더 조회 후 직접 안내 완료"
```

**시나리오 흐름 (중요한 건 — 컨펌 필요):**
```
정현수 팀장 → 봇 DM
  "대표님께 이번 달 플랫폼 개발 인력 1명 충원 건 허가 요청드리고 싶습니다."

봇 → 대표님 DM (컨펌 요청)
  "🔔 [컨펌 요청]
   정현수 팀장이 문의했습니다:
   '이번 달 플랫폼 개발 인력 1명 충원 건 허가 요청'
   → 어떻게 회신할까요?"

대표님 → 봇
  "다음 달에 다시 얘기하자고 해줘"

봇 → 정현수 팀장 DM
  "안녕하세요 정현수 팀장님, 대표님 말씀으로는
   '다음 달에 다시 논의하자'고 하셨습니다."

봇 → 대표님 DM
  "✅ 정현수 팀장님께 답변 전달 완료했습니다."
```

**구현 포인트:**
- 중요도 판별 기준 (Gemini 분류): 인사·예산·승인·대외비 키워드 → 컨펌 루트
- 직원별 대화 세션 컨텍스트 유지 (멀티턴)
- 민감 정보(급여·M&A 등) 필터는 기획서 §6 기준 적용

### S3. 관심 사안 뉴스 검색 및 브리핑

**개요:** 회사 주요 관심 분야 키워드로 뉴스를 검색해 대표님께 브리핑한다.

**시나리오 흐름:**
```
대표님 → 봇
  "요즘 GNSS·RTK 시장 동향 뉴스 좀 찾아줘"

봇 (뉴스 검색 후)
  "📰 [GNSS·RTK 시장 동향 — 2026-04-09 기준]

   1. [제목] 국토부, 초정밀 측위 인프라 확대 사업 입찰 공고
      요약: ...
      출처: OO일보

   2. [제목] LG유플러스, U+ 초정밀측위 OEM 파트너십 확대
      요약: ...
      출처: OO뉴스

   ※ 추가 키워드 지정 또는 정기 브리핑 설정 가능합니다."
```

**관심 키워드 (초기값, 기획서 §1·§3.4 기준):**
- GNSS, RTK, 초정밀측위, HI-RTK
- IoT 안전 장구, AI 비전 엣지
- 국토부 측위 인프라, U+ OEM

**구현 포인트:**
- Web Search API 연동 (Gemini 또는 외부 뉴스 API)
- 정기 브리핑(예: 매주 월요일 오전) 크론 설정 — Phase 2
- 키워드는 요구사항.md 또는 대화로 업데이트 가능하게

### S4. 일정 요약 보고 (캘린더 + 슬랙 컨텍스트)

**개요:** Google Calendar + `#noti-업무일정` 채널을 종합해 일정을 요약하고, 누가·언제·어디서를 명확히 보고한다.

**시나리오 흐름:**
```
대표님 → 봇
  "오늘 일정 요약해줘"

봇 (Calendar API + #noti-업무일정 파싱 후)
  "📅 [2026년 4월 9일 (목) 일정 요약]

   1. 10:00 ~ 11:00 | CXO 주간 보고
      장소: 본사 2층 회의실
      참석: 구자덕 대표, 이종혁 본부장

   2. 14:00 ~ 15:00 | HI-CCP 납품 현황 점검
      참석: 김봉석 팀장, 이창현 프로

   3. 종일 | [외근] 이지민 팀장

   ※ #noti-업무일정 원문 기반 보완 포함."
```

**구현 포인트:**
- 현재 v15.7 일부 구현(조회·파싱) — 자동 브리핑(크론)은 미구현
- 참석자 이메일 → HNI 멤버 이름 변환 (`resolveEmailsInText`)
- 장소·비고 포함 여부 판단 후 출력
- 매일 오전 자동 DM 브리핑 — Phase 2 목표

### S5. 경영 채널 재무 현황 요약 보고

**개요:** `#cmm-cxo` 채널 히스토리를 분석해 재무·경영 현황을 요약 보고한다.

**시나리오 흐름:**
```
대표님 → 봇
  "이번 달 경영 채널 재무 현황 요약해줘"

봇 (#cmm-cxo 히스토리 분석 후)
  "💼 [경영 현황 요약 — 2026년 4월]

   ▶ 매출
   - 이번 달 HI-RTK 납품 계약 2건 체결 (OO기관, OO사)
   - 누적 수주액: 전월 대비 약 15% 증가 언급

   ▶ 비용·이슈
   - 서비스지원팀: 현장 유지보수 출장비 증가 보고
   - 플랫폼팀: 서버 비용 최적화 검토 중

   ▶ 주요 결정 사항
   - 다음 달 CXO 보고 일정 조율 중

   ※ 채널 원문 기반 — 수치는 원문 그대로 인용."
```

**구현 포인트:**
- 현재 v15.7 부분 구현 (`report_management_status` finance 트랙)
- Gemini 요약 품질 개선: 수치/날짜 명확 인용, 추측 금지
- 주간/월간 정기 보고 자동화 — Phase 2~3 목표

### S6. 채널별 업무 맥락 파악 및 요약 보고

**개요:** 각 부서/운영 채널의 최근 흐름을 파악해 채널별 업무 현황을 요약한다.

**시나리오 흐름:**
```
대표님 → 봇
  "각 팀 채널 업무 현황 요약해줘"

봇 (채널별 히스토리 분석 후)
  "📊 [채널별 업무 현황 요약]

   ▶ #cmm-영업지원 (영업)
   - 신규 고객사 견적 문의 2건 진행 중
   - OO지자체 납품 건 계약 검토 단계

   ▶ #dep-상품관리팀
   - HI-PPE v4.1 스펙 정의 회의 완료
   - 출시 일정 5월 초로 조율 중

   ▶ #ops-제품납품
   - 이번 주 납품 3건 완료, 1건 지연(물류사 사유)

   ▶ #기술팀
   - RTK 기준국 신호 불안정 이슈 발생 → 김찬영 프로 대응 중

   ※ 요약 기준: 각 채널 최근 50~100개 메시지"
```

**구현 포인트:**
- 현재 v15.7: 3채널만 연결 (`cmm-cxo`, `cmm-영업지원`, `noti-업무일정`)
- 나머지 채널 연결은 기획서 §5.1·로드맵 기준 Phase 3 목표
- 채널별 요약 도구 enum 확장 필요 (`report_management_status` categories 추가)
- 요약 주기: 요청 시 즉시 + 주간 자동 보고(Phase 3)
