## 1. 개요

구대표집사봇은 **구대표님 전용 AI 비서**를 중심으로, Slack 안에서 일정·업무·부서 협업부터 직원 상담·기술 질의응답까지 **단계적으로** 통합 처리하는 것을 목표로 합니다.

| 항목 | 내용 |
|------|------|
| 봇 이름 | @구대표집사봇 |
| 주요 사용자 | 구자덕 대표님 (CEO) — 전용 AI 비서 |
| 플랫폼 | Slack — DM 및 전용 채널 |
| AI 엔진 | Gemini API (Google) — 현재 구현: `gemini-2.5-flash` |
| 현재 버전 | v15.4 (`api/slack.js`) |
| 호스팅 | GitHub(소스) + Vercel(서버리스) |
| 연동 서비스(목표) | Google Calendar · Google Drive · Slack API |
| **현재 구현 연동** | Slack Web API · Google Calendar API(읽기, JWT 서비스 계정) · Gemini — **Google Drive 미연동**, **슬래시 명령 미구현** |

## 2. 역할 및 채널 구조

4개 역할로 구분하고, 각 역할마다 전용 Slack 채널 또는 DM을 배정합니다.

| 역할 | Slack 채널 / DM | 봇의 주요 기능(목표) |
|------|-----------------|----------------------|
| 1 · 일정·업무 | DM + `#noti-업무일정` + `#jj-프롬프트`(대표 명령 권장) | 일정 조회·등록, 오늘 브리핑, 할 일·리마인더 등 |
| 2 · 부서업무 | 부서·CXO 채널(아래 매핑 참고) | 지시 전달·요약, 진행 취합·보고, 마감 관리 등 |
| 3 · 직원상담 | DM(비공개) | 상담, 요약 후 대표 보고(민감 정보 제외) |
| 4 · 기술상담 | DM + 기술 관련 채널 | GNSS·RTK·HI-RTK 등 기술 Q&A, 문서 기반 답변 |

**현재 구현:** `BOSS_USER_ID`(구대표님) 메시지에 한해, Gemini + 함수 호출로 **경영/영업/일정 3트랙 채널 히스토리·캘린더(읽기)** 분석 및 **지정 직원 DM 전송**까지 동작합니다. 역할 2~4의 전 범위·자동화·Drive 연동은 **미구현**입니다.

## 3. 역할별 상세 기능

### 3.1 역할 1 — 일정·업무 관리

**채널:** DM + `#noti-업무일정` 등 · **대상:** 구대표님 중심

| 기능 | 상태 |
|------|------|
| Google Calendar **양방향** 연동(조회·등록·수정·삭제) | 목표 — **현재는 조회(읽기) 위주** |
| 매일 오전 「오늘 일정 브리핑」 자동 DM | 목표 |
| 회의 30분 전 리마인더 | 목표 |
| `/오늘일정`, `/다음회의`, `/할일추가` 슬래시 명령 | 목표 |
| 주간 일정 요약(월요일 오전) | 목표 |
| 할 일(To-Do) 관리·마감 알림 | 목표 |

### 3.2 역할 2 — 부서업무 관리

**채널:** `#dep-*`, `#cmm-cxo` 등 · **대상:** 대표 ↔ 직원

- 업무 지시 정리·요약 후 채널 전달, 진행 현황 취합·보고, 마감·기한 알림, 회의록·액션아이템 정리, 지시 히스토리 검색 등 **목표 스펙**.
- **현재 구현:** `finance`→`#cmm-cxo`, `sales`→`#cmm-영업지원`, `calendar`→`#noti-업무일정`(+구글 캘린더 읽기) **3트랙만** 연결. `#dep-*`, `#cmm-경영본부`, `#cmm-제품개발` 등 §5.1의 나머지 부서·기술·공지 채널은 **코드 미연결**(로드맵).

### 3.3 역할 3 — 직원 상담

**채널:** DM(비공개) · **대상:** 전 직원

- 직원 → 봇 DM 접수, 상담, 종료 후 대표 보고, **급여·M&A·투자 협상 등 민감 정보 제외** 필터, 직원 안내 메시지 등 **목표 스펙**.
- **현재 구현:** 일반 직원용 DM 상담 파이프라인 **없음**(대표 전용 `handleBoss`만 동작).

### 3.4 역할 4 — 기술 상담

**채널:** DM + `#기술팀` 등 · **대상:** 대표 + 직원

- GNSS·RTK·초정밀측위, HI-RTK, U+ 초정밀측위 OEM, **Google Drive 기술 문서** 연동, 답변 불가 시 **이종혁·김봉석·김찬영** 등 담당자 에스컬레이션 **목표**.
- **현재 구현:** 전용 기술 FAQ·Drive RAG·에스컬레이션 **없음**. 시스템 프롬프트에 회사 핵심 기술 키워드가 포함되어 있으나, 역할 4 전용 플로우는 아님.

### 7.1 데이터 흐름(목표)

사용자 → Slack(DM / 전용 채널) → 봇 엔진(Gemini 등) → **역할·권한 판별** → 기능 실행 → 응답 반환.

**현재 구현:** Slack Events → Vercel `POST` 핸들러 → 서명 검증 → (대표만) Gemini + 선택적 Slack·Calendar 호출 → 스레드/채널 응답.

### 7.2 연동 시스템

| 시스템 | 연동 목적 | 현재 구현 |
|--------|-----------|-----------|
| Slack API | 수신·발신, DM, 채널, (향후) 슬래시 명령 | 메시지 이벤트·`chat.postMessage`·`conversations.history` |
| Gemini API | 자연어, 요약, 상담·기술 응답 | 대표 전용 브리핑·분석 루프 |
| Google Calendar | 일정 CRUD, 리마인더, 브리핑 | **읽기(이벤트 조회)** 만 |
| Google Drive | 기술·기획 문서 검색 | **미연동** |
| 사내 지식베이스 | GNSS·HI-RTK FAQ 등 | **미연동**(프롬프트 내 정적 설명 수준) |

### 7.3 기술 스택

| 구분 | 기획(방향) | 현재 저장소 |
|------|------------|-------------|
| 봇 런타임 | Node.js + Slack Bolt SDK 등 | Node.js 24 ESM, **단일 서버리스 핸들러**(`fetch` 기반, Bolt 미사용) |
| AI | Gemini 2.5 Flash | 동일 |
| 캘린더 | Calendar API v3 | JWT 서비스 계정, `calendar.readonly` |
| 문서 | Drive API v3 | 없음 |
| 호스팅 | GitHub + Vercel | 동일 |
| 보안 | Signing Secret, OAuth, 환경변수 | Signing Secret + 환경변수, Raw Body로 서명 검증 |

## 8. 환경 변수 (Vercel)

| 변수명 | 설명 |
|--------|------|
| `SLACK_BOT_TOKEN` | Slack Bot OAuth Token |
| `SLACK_SIGNING_SECRET` | Slack 요청 검증 Secret |
| `BOSS_USER_ID` | CEO 전용 분기용 Slack User ID (예: `U02M1T5E1N3`) |
| `GEMINI_API_KEY` | Google Gemini API Key |
| `GOOGLE_CLIENT_EMAIL` | 서비스 계정 이메일 |
| `GOOGLE_PRIVATE_KEY` | 서비스 계정 Private Key(`\n` 이스케이프) |
| `GOOGLE_CALENDAR_ID` | 대표님(또는 조회 대상) 캘린더 ID |

### 알려진 이슈(인증·코드)

- 서비스 계정 이메일 환경변수는 **`GOOGLE_CLIENT_EMAIL`** 을 사용합니다(`GOOGLE_SERVICE_ACCOUNT_EMAIL` 등 혼용 금지).
- 캘린더 JWT 발급 함수명은 **`getGoogleAccessToken`** 과 호출부가 일치해야 합니다(v15.4 기준 정렬됨).

### 10.2 Slack 앱 설정(요약)

- **Event Subscriptions:** Request URL → Vercel 배포 URL + `/api/slack` 경로(실제 라우트는 Vercel이 `api/slack.js`에 매핑).
- 구독 이벤트 예: `message.channels`, `message.groups`(비공개 채널 사용 시), DM이면 `message.im` — **봇이 해당 채널에 초대**되어 있어야 `conversations.history`가 동작합니다.
- **OAuth 스코프(예):** `chat:write`, 채널 히스토리 읽기에 맞는 `channels:history` / `groups:history` / `im:history` 등(워크스페이스 정책에 맞게 조정).
- **Signing Secret** → `SLACK_SIGNING_SECRET`, **Bot Token** → `SLACK_BOT_TOKEN`.
